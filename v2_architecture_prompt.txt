# MISSION
You are an Elite Backend & ML Engineer. Your task is to build a highly scalable, modular backend API for an IELTS Audio Assessment application from scratch. We are discarding an old, flawed prototype and building the "V2" architecture. Your goal is to write clean, well-documented, production-ready Python code (FastAPI + Celery) focused entirely on the backend orchestration, machine learning inference, and LLM evaluation. No frontend code is required.

# THE MISTAKES OF V1 (WHAT NOT TO DO)
1. **The FFmpeg Silence Mistake:** V1 used ffmpeg silencedetect to find pauses. This failed because background noise prevented the volume from dropping. **Solution:** We will use a neural VAD (Voice Activity Detection).
2. **The GoP Mistake:** V1 used a wav2vec2-TIMIT model for Goodness of Pronunciation. It penalized non-American accents heavily. **Solution:** We will use ASR Confidence as a proxy for intelligibility.
3. **The Sequential VRAM Bloat:** V1 loaded Whisper and Wav2Vec2 simultaneously, crashing 12GB GPUs. **Solution:** We will use one unified model family (WhisperX) that handles VAD, ASR, and Alignment in a single pass.
4. **Hardcoded Heuristic Scoring:** V1 tried to calculate IELTS bands using hardcoded math. **Solution:** Python will ONLY calculate raw metrics. We pass these metrics to Gemini to determine the actual IELTS band scores based on the official rubric.

# THE NEW ARCHITECTURE (V2)
1. **Framework:** FastAPI (REST API) + Celery (Task Queue) + Redis (Broker).
2. **Audio Processing:** WhisperX (loads Whisper + PyAnnote VAD + Phoneme Aligner).
3. **LLM Evaluator:** Google Gemini (Gemini 3 Flash / 2.5 Flash).
4. **Design Pattern:** Use the Strategy Pattern. Create interfaces for ASRProvider and PronunciationProvider. Implement the WhisperX versions. Leave stubs for future providers (like Azure or NeMo). All thresholds configurable via .env.

# PIPELINE EXECUTION FLOW
1. **Prep:** Convert audio to 16kHz, mono WAV.
2. **VAD & ASR (WhisperX):** Run VAD, transcribe, and run Forced Alignment to get ms-accurate timestamps and word-level confidences.
3. **Metric Calculation:** Calculate WPM, speech ratio, list of pauses (gaps > 0.4s between VAD segments). Calculate TTR. Find ratio of words with ASR confidence < 0.60 (flag these as intelligibility errors).
4. **LLM Evaluation (Gemini):** Inject Transcript, Fluency Metrics, Lexical Metrics, and Pronunciation errors into Gemini. Force structured JSON output.
5. **Save & Return:** Save compiled JSON report.
